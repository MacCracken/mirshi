# Security audit — 2026-06-30 (0.7.0 sandbox-escape sweep)

**Scope**: mirshi as a sandbox-class trust boundary — it `fork`+`exec`s a foreign-ish
agnos static ELF and runs it under `PTRACE_SYSCALL`, translating its agnos-ABI
syscalls while holding host privilege (a classic **confused-deputy** surface). This
is the 0.7.0 roadmap sweep of the escape classes.

**Method**: adversarial multi-auditor review (five specialists: path-escape, seccomp
default-deny, TOCTOU/child-memory, arg-confusion, supervisor-integrity), each reading
the live source and constructing concrete PoCs. Headline findings empirically
confirmed against the built binary on x86_64 Linux. Surface reviewed: `src/intercept.cyr`,
`src/dispatch.cyr`, `src/translate.cyr`, `src/scratch.cyr`, `src/seccomp.cyr`,
`src/limits.cyr`, `src/decode.cyr`, `docker/`.

## Executive summary

| Class | Verdict |
|---|---|
| (a) TOCTOU / child-memory | **Safe by design** — single-threaded, ptrace-stopped, single-process child is classic-TOCTOU-safe; child-controlled output pointers are kernel-bounded to the child's own address space. |
| (b) seccomp default-deny | **Default-deny proven** (BPF skeleton correct, terminal `RET_KILL_PROCESS`). Residual completeness gaps: fail-**open** install, x32 bit unmasked, args unconstrained — all hardened / tracked here. |
| (c) **path-translation escape** | **BLOCKER class** — mirshi has **zero in-supervisor path confinement**. Every path-bearing call reaches arbitrary host paths in the bare CLI. |
| (d) arg-confusion / translation integrity | **Mostly clean** — `mmap` synthesis hardened (no PROT_EXEC/MAP_FIXED/file-backed); residual minors: mmap-address ASLR infoleak, world-readable create modes. |
| supervisor integrity | **Clean** — no out-of-bounds of supervisor buffers, no infinite loop, no hijack; one-slot carry-state safe given the single-process child. |

**32 findings: 5 blocker, 4 major, 13 minor, 1 nit, 9 info (22 reachable).** The
entire blocker tier is **class (c) path-escape**. The other four classes are sound;
their residual items are minor/bounded hardening, addressed in this milestone.

## Confinement model (load-bearing context)

mirshi's M2 filesystem translation uses **transparent path pass-through**: the agnos
path is staged verbatim into the child and handed to the host `open(2)`/`stat(2)`/etc.,
resolved in the **supervisor's own filesystem view** — no `chroot`, no mount namespace,
no canonicalization, no rootfs prefix (`src/scratch.cyr:61` `stage_at`; the only check
is `namelen <= PATH_MAX`). The agnos child runs at the host uid (no privilege drop).

Confinement therefore depends entirely on the **deployment**:

- **Docker v1 vehicle** (`docker/`, `FROM scratch`, no host bind mounts) — the
  container **mount namespace IS the rootfs boundary** (the roadmap's stated model).
  The audit confirms it bounds every path escape to the image contents: an absolute
  path resolves at the image root, `..` is kernel-clamped to the mount-NS root, and a
  cross-device `link(2)` to a host inode fails `-EXDEV`. **In the v1 vehicle the path
  escapes are contained.**
- **Bare CLI** (`mirshi <elf>` run directly, no container) — **nothing** confines the
  child; it has full host-FS access at the operator's uid. This is a dev/test mode,
  not the v1 product, but it is a sharp footgun.

**Empirically confirmed (this audit):** a bare-CLI agnos fixture issuing
`open#7("/abs/host/secret", …)` + `read#5` + `write#1` exfiltrated a host file placed
outside any intended root. PoC reproduced on the built binary.

## Findings by class

### (c) Path-translation escape — BLOCKER class · *fix deferred to the confinement milestone*

All reachable in the bare CLI; all bounded to the image in the Docker vehicle.

| # | Finding | Sev | Op |
|---|---|---|---|
| c1 | Absolute-path read of arbitrary host files (`/etc/shadow` class) | blocker | `open#7` |
| c2 | `..` traversal escapes any future rootfs prefix (no canonicalization) | blocker | all path ops |
| c3 | `link#32` **hardlink** any readable host file into a writable location | blocker | `link#32` |
| c4 | `AO_CREAT` + `AO_TRUNC` create/truncate arbitrary host files (the create *mode* is tightened to 0600 this milestone; the arbitrary *path* is the blocker, deferred) | blocker | `open#7` |
| c5 | `unlink#30` / `rmdir#10` / `rename#31` destroy or move arbitrary host paths | blocker | unlink/rmdir/rename |
| c6 | `open#7` follows symlinks (no bounded resolution) | major | `open#7` |
| c7 | `open#7` of `/proc/self/mem`, `/proc/<pid>/mem` — memory R/W primitive | major | `open#7` |
| c8 | `open#7` of device files (`/dev/sda`, `/dev/mem`) — raw access if mirshi is root | major | `open#7` |
| c9 | `getdents#29` enumerates any opened host directory | major | `getdents#29` |
| c10 | `stat#33` host-file metadata oracle | minor | `stat#33` |
| c11 | Embedded-NUL / explicit-length path mismatch (load-bearing once confinement lands) | minor | `stage_at` |

**Root cause**: `stage_at` (`src/scratch.cyr:61`) stages the path with no resolution or
confinement; the handlers (`src/dispatch.cyr`) feed it to plain `open(2)`/`stat(2)`/etc.
**The fix** (next milestone): a supervisor-resolved **rootfs** — open a configured root
once (`rootfd`), and route every path op through kernel-enforced bounded resolution
(`openat2` with `RESOLVE_IN_ROOT`/`RESOLVE_BENEATH` for `open`; parent-dir-anchored
`*at` ops for `mkdir`/`unlink`/`rename`/`link`/`stat`/`getdents`), rejecting any escape
before staging. This is unprivileged (no `CAP_SYS_CHROOT`), TOCTOU-safe (kernel resolves
atomically), and adds defense-in-depth on top of the container NS. Tracked as the next
milestone (see [roadmap](../development/roadmap.md)); the user-selected plan is
**harden now, confine next**.

### (b) seccomp default-deny — proven, with completeness hardening (this milestone)

**Default-deny proof (confirmed).** The classic-BPF program (`src/seccomp.cyr`
`apply_child_bound`) is structurally correct: arch-guard (`AUDIT_ARCH_X86_64`, wrong
arch → `KILL`), load
`nr`, a JEQ chain whose `jt` offset (`na - k`) lands every match on the terminal
`RET_ALLOW`, and a **terminal `RET_KILL_PROCESS`** for any non-match. `PR_SET_NO_NEW_PRIVS`
is set first; install order is `TRACEME` → rlimits → seccomp → `execve` (so the bound
persists across exec and `prlimit64`/`seccomp`/`prctl` are themselves *not* allowlisted
→ unraisable by the child). Under `PTRACE_SYSCALL` every syscall is trapped+rewritten
*before* seccomp evaluates the **output** number (ADR 0004), so there is no
"around-the-translation" path. The child cannot reach a non-allowlisted host syscall.

Residual completeness items (content, not skeleton):

| # | Finding | Sev | Status |
|---|---|---|---|
| b1 | seccomp/rlimit install failure is silently **fail-open** (child runs unbounded) | minor (reachable) | **fixed this milestone** (fail-closed) |
| b2 | x32-ABI bit (`0x40000000`) unmasked — x32 number could alias an allowlisted one | minor (bounded) | **fixed this milestone** (mask → `KILL`) |
| b3 | `execve(59)` allowlisted for the child's whole lifetime, not just initial load | minor (bounded) | documented bounded (agnos has no execve; never re-emitted) |
| b4 | `open(2)`/`mmap(9)` allowlisted with no arg/PROT constraint | minor (bounded) | deferred defense-in-depth (mirshi pins the args; arg-JEQ tracked) |

### (a) TOCTOU / child-memory — safe by design

- **No TOCTOU.** The child is single-threaded, single-process (no `clone`/`fork` in the
  allowlist; `spawn#3` is `ENOSYS`), and ptrace-**stopped** in-kernel while the supervisor
  stages/reads/writes its memory. There is no window for concurrent arg mutation between
  check and act. `FLAG_CONTINUE` (the seccomp-notify TOCTOU 0-day class, ADR 0005) is
  **moot** — mirshi is ptrace-only and never `CONTINUE`s a security-relevant syscall.
- **Child-controlled output pointers are bounded.** The exit-stop repack `pvm_write`s to
  `_m2_out_ptr` (the child's own output pointer) and the red-zone scratch is derived from
  the child's `rsp` — but `process_vm_writev(311)` is kernel-confined to the child's **own**
  address space, so the child can only name memory it already owns (self-corruption, not
  escape). The enter→exit carry-state cannot be desynchronized: the `at_entry` phase flag
  flips only on a syscall-stop, never on a signal-stop.

### (d) Arg-confusion / translation integrity — mostly clean

- **`synth_mmap_regs` hardened** (`src/translate.cyr:173`): all five synthesized registers
  are supervisor constants — `PROT_READ|PROT_WRITE` (no `PROT_EXEC`), `MAP_PRIVATE|MAP_ANONYMOUS`
  (no file-backing, no `MAP_FIXED`), `fd = -1`, `addr = 0`. The child influences only the
  length (2 MB-rounded, `RLIMIT_AS`-capped). No way to obtain an executable, file-backed, or
  fixed-address mapping.
- Residual minors: `mmap` returns a real host userspace VA to the child (**host-ASLR
  infoleak**, informational — bounded by NS/seccomp/rlimit, not directly exploitable);
  `open`/`mkdir` synthesize **world-readable** modes 0644/0777 — **tightened to 0600/0700
  this milestone** (least-privilege; agnos carries no mode). `dup#8` grants a real fd where
  agnos defines a no-op stub (bounded to the child's own fd table). `getdents` truncates
  `d_ino` to 32 bits (faithful to the agnos ABI).

### Supervisor integrity — clean

No reachable way for the child to corrupt or hijack the supervisor: the repack reads
child-controlled lengths into supervisor buffers but every length is bounded (`namelen <=
PATH_MAX`, the getdents `cap`/`avail` clamps, the `d_reclen < 1` anti-spin guard); the
`getdents_repack` `strlen` is bounded by the host kernel's NUL-termination contract; the
two-path staging writes only to **child** stack memory; the one-slot carry-globals are safe
given the single-process child. `--no-seccomp` / `--selftest-trace` are operator flags, not
child-reachable.

## Remediation status (this milestone — "harden now")

- **Fail-closed bound** — `apply_child_bound`/`apply_child_rlimits` return status is now
  checked in `_child_exec`; in bounded mode a failed seccomp install **aborts the child**
  (diagnostic + `exit`) rather than running it unbounded. Filter `alloc()`s guarded.
- **x32 mask** — the seccomp program rejects any `nr >= 0x40000000` (`KILL`) before the
  allowlist JEQ chain, closing the x32-bit ambiguity permanently.
- **Least-privilege create modes** — `open#7` 0644 → 0600, `mkdir#9` 0777 → 0700.
- **This audit document** + the roadmap entry scheduling the supervisor rootfs confinement.

## Deferred to the next milestone ("confine next")

- **Supervisor-resolved rootfs confinement** (the class-c blocker fix): `--root` +
  `openat2`/`*at` bounded resolution for all seven path handlers. Until it lands, the
  **container mount namespace is the confinement boundary** and the **bare CLI is
  unconfined by design** — documented here and in the roadmap.
- Defense-in-depth seccomp **arg** constraints (b4: assert `mmap` `PROT` has no `PROT_EXEC`).
- The escape-attempt fault-harness containment cases land **with** the confinement work
  (they cannot pass in the bare CLI until confinement exists; in-container containment is
  asserted by the Docker smoke gate).

## 0.7.1 remediation progress — class-(c) path-escape

- **Bite 1 (done)** — `--root <dir>` confines `open#7` via `openat2 RESOLVE_IN_ROOT`
  anchored at a per-child rootfd ([ADR 0009](../adr/0009-rootfs-confinement-openat2-in-child.md)).
  Contained under `--root`: c1 (abs read), c2 (`..`), c4 (create), c6 (symlink),
  c7 (`/proc/*/mem`), c8 (device files), c9 (getdents) — every `open`-and-fd-based vector.
- **Bite 2 (done)** — `mkdir`/`rmdir`/`unlink`/`rename`/`link`/`stat` confined via the
  `*at` family anchored at the rootfd with a lexically-sanitized relative path
  (`sanitize_rootrel`: strip leading `/`, reject `..`). Contained under `--root`:
  **c3 (hardlink), c5 (unlink/rmdir/rename), c10 (stat)**. So under `--root` the entire
  class-(c) path surface is confined (verified by `scripts/it/confine.sh`). Residual:
  a **pre-existing** symlink in the rootfs can redirect a mutation op (lexical
  sanitization is not symlink-aware) — but agnos has no symlink syscall, so this needs a
  hostile rootfs (rootfs-trust, narrower than the blocker tier); `open` has no such
  residual (`RESOLVE_IN_ROOT` is symlink-safe).
- **Bite 3 (next)** — activate `--root /` in the Docker vehicle's `ENTRYPOINT` + an
  in-container escape-attempt assertion in the smoke gate.
